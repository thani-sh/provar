# 002 - Graph-Based Interaction Model

## Context

Linear test scripts do not represent the branching reality of user journeys, making complex flows hard to visualize and manage.

## Decision

We will use an **Infinite Canvas** and **Graph-Based Logic** as the primary interaction model for defining tests.

## Consequences

- Clear visualization of all possible user paths.
- Intuitive management of branching and decision points.
- Foundation for hierarchical/nested test structures.
