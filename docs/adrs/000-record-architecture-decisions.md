# 000 - Record Architecture Decisions

## Context

We need to record architectural decisions made during the development of Provar.

## Decision

We will use Architecture Decision Records (ADRs) to document significant architectural choices. ADRs will be stored in `docs/adrs` as Markdown files.

Since we use Git-flow, the status of an ADR is determined by its presence in the repository:

- Merged into the main/develop branch means **Accepted**.
- Open Pull Request means **In Review**.

## Consequences

- Architectural history is preserved and searchable.
- Future developers understand the "why" behind decisions.
