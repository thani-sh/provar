# Provar Project TODOs & Action Items

This document tracks the backlog of engineering tasks required to bring Provar from its current skeleton implementation to its designed production state, as specified in the ADRs and PDRs.

---

## 1. `@libs/compiler` (Compiler Library)

- [ ] **`TODO-COMPILER-001`**: **Replace mock translation with AI-based code generation pipeline (`ADR-003`)**
  - **Description**: Replace the `translateNodeToCode` rules-based fallback placeholder (which matches hardcoded strings like `"login"` or `"create list"`) with a production AI code generator.
  - **Action**: Use the `@libs/agents` session to pass the resolved runtime `GroundingContext` (HTML DOM strings and base64 screenshots) to a Gemini model to dynamically synthesize robust, selector-accurate Playwright code.
- [ ] **`TODO-COMPILER-002`**: **Implement context-aware error healing compiler loop (`ADR-003`)**
  - **Description**: Add a self-correction compilation step.
  - **Action**: If a generated action fails execution during the grounding compiling phase, parse the console logs and locator error output, feed the failure state back to the agent session, and attempt auto-correction.

---

## 2. `@libs/executor` (Runtime Test Engine)

- [ ] **`TODO-EXECUTOR-001`**: **Implement Visual Comparison & Regression Engine (`ADR-009`, `PDR-009`)**
  - **Description**: Enable automatic visual validations at runtime.
  - **Action**: Take post-action screenshots for every step in the path, use `pixelmatch` to validate transient screenshots under `current/` against the baseline in `accepted/`, and write mismatch overlays to a `diff/` directory when regression thresholds are exceeded.
- [ ] **`TODO-EXECUTOR-002`**: **Implement Black-and-White Ignore Masking (`ADR-009`)**
  - **Description**: Exclude dynamic or high-churn areas (dates, timestamps, third-party widgets) from failing visual assertions.
  - **Action**: Check for matching mask images under `masks/` and paint over `#000000` regions in both current and baseline screenshots with a solid uniform color before matching.
- [ ] **`TODO-EXECUTOR-003`**: **Implement Deterministic Path ID Slug Resolution (`ADR-010`, `PDR-010`)**
  - **Description**: Derive stable, prefix-stripped namespace slugs to group baseline visual assets cleanly.
  - **Action**: Dynamically compute prefix-stripped action IDs joined by hyphens (e.g., `v2b3n-k1l2m`) to isolate and namespace screenshot directories under `.provar/screenshots/<state>/<test-path>/<path-slug>/`.
- [ ] **`TODO-EXECUTOR-004`**: **Support Step-Level `visualCompare` Option (`PDR-009`)**
  - **Description**: Support selective regression checks per step.
  - **Action**: Update the `Action` interface and `action()` helper options in `libs/executor/src/index.ts` to accept the `visualCompare: boolean` parameter generated by the compiler, allowing steps to skip visual diff failures when toggled off.
